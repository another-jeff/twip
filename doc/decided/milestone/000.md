# Milestone 0: Open Door

Milestone 0 proves the smallest useful Twip loop.

## Goal

Support one parser command:

```text
open door
```

The command should locate a wooden door in the world, apply deterministic behavior, mutate world state, return a result, and pass regression tests.

## Required pieces

```text
World
Entity
Component
Action
Result
Parser
Door extension
door_wooden preset
open door regression test
```

## Target flow

```text
text input
→ parser
→ Action
→ World target lookup
→ Entity
→ Component
→ state change
→ Result
→ regression test
```

## Initial world

```text
World:
  Entity:
    key: door_wooden
    name: wooden door
    Component:
      Door
        is_open: false
```

## Expected behavior

When the player enters:

```text
open door
```

Then:

```text
result.succeeded == true
door.is_open == true
result.message == "You open the wooden door."
```

## Repeated command behavior

When the wooden door is already open and the player enters:

```text
open door
```

Then:

```text
result.succeeded == true
door.is_open == true
result.message == "The wooden door is already open."
```

## Invariant

World state is the source of truth.

The result message is useful feedback, but the real proof is the state of the world after the command runs.
