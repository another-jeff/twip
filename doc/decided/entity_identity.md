# Entity Identity and Parser Names

Entity identity is not language.

An entity has a unique internal key assigned by the world. Parser-facing names and traits are not unique.

```text
Entity key = generated internal identity
Entity name = primary parser/display name
Entity names = parser-facing nouns
Entity traits = parser-facing disambiguators
Preset name = reusable construction recipe
```

These are different things.

```text
preset: door_wooden
generated key: entity_1
names: door
traits: wooden, north
```

Callers do not assign entity keys.

```text
Caller describes entities.
World identifies entities.
Parser resolves language to entities.
```

An entity is created without a key.

```text
Entity(names=("door",), traits={"wooden", "north"})
```

The world assigns a key when the entity is added.

```text
world.add(entity)
  -> entity.key = entity_1
```

The key is internal state identity. It is not parser language.

Multiple entities may share the same parser name.

```text
entity_1
  names: door
  traits: north, wooden

entity_2
  names: door
  traits: south, wooden
```

The parser must not assume that a name resolves to one entity.

Target lookup returns one of three outcomes:

```text
none
one
many
```

Examples:

```text
open door
  -> matches entity_1 and entity_2
  -> ambiguous

open north door
  -> matches entity_1
  -> resolved

open trapdoor
  -> matches nothing
  -> missing target
```

Ambiguity is a normal result, not an exception.

```text
Which door do you mean?
```

Names and traits have different jobs.

```text
names = nouns/parser handles
traits = disambiguating words
key = generated identity
```

Example:

```text
names: door
traits: north, wooden
```

This matches:

```text
door
north door
wooden door
north wooden door
```

This does not match:

```text
north
wooden
```

A target phrase must include at least one parser name. Traits alone do not identify an entity.

Preset names do not become entity keys.

```text
door_wooden
  preset function

Entity(names=("door",), traits={"wooden"})
  entity produced by preset

world.add(entity)
  generated world identity
```

Rules decided:

```text
Entity keys are generated by World.
Entity keys are not constructor arguments.
Entity keys are not parser-facing.
Parser matching uses names and traits only.
Names do not need to be unique.
Traits do not need to be unique.
Preset names do not become entity keys.
Target lookup returns candidates.
Many candidates produce an ambiguous Result.
Ambiguity does not mutate world state.
```

Rooms and containment are not required for entity identity.

The current implementation may search all world entities. Later, room containment can narrow parser scope.

Future direction:

```text
World contains rooms/entities.
Room contains visible/reachable entities.
Parser candidates come from current scope.
```

But the identity rule remains the same:

```text
World assigns identity.
Parser never matches identity.
```
